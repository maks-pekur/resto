# ADR 0000: Template

- **Status:** template
- **Date:** YYYY-MM-DD
- **Deciders:** <list>
- **Supersedes:** —
- **Superseded by:** —

## Context

What is the problem being solved? What forces are at play (technical,
organizational, regulatory, cost)? What constraints frame the decision?

## Decision

The chosen approach, stated unambiguously in one or two sentences.

## Alternatives considered

For each serious alternative: a brief description, the strongest argument
for it, and the reason it was not chosen here. Do not list throwaway
options — only candidates a reasonable engineer might pick.

## Consequences

### Positive

- What gets easier or safer because of this decision.

### Negative

- What gets harder or riskier. (Every real decision has tradeoffs — if
  you cannot list any, you have not understood the alternatives.)

### Neutral

- Things that change but are neither better nor worse.

## Implementation notes

Concrete pointers: which packages/files implement this, links to docs,
runbook entries, anything that helps someone find the implementation
later.
